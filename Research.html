Black Hole Attack Analysis in AODV

A comprehensive study on network security vulnerabilities in ad-hoc wireless networks

Back to Portfolio

Abstract

This research project investigates the Black Hole attack in Ad-hoc wireless networks using the AODV (Ad-hoc On-Demand Distance Vector) routing protocol. The simulation was conducted using ns-3 (Network Simulator 3) on Parrot OS, with NetAnim for visualization. The study demonstrates how a malicious node can disrupt network communication by dropping all incoming packets, effectively creating a "black hole" in the network topology.

Research Metrics

  • Simulator: ns-3 (v3.41)
  • Protocol: AODV
  • Network Type: Ad-hoc WiFi
  • Nodes: 5 (Linear Topology)
  • Attack Type: Black Hole
  • Visualization: NetAnim 3.109

2.2 Black Hole Attack Implementation

The core mechanism of the attack is implemented through a custom receive callback function that intercepts and drops all incoming packets at the malicious node:

blackhole-attack.cc (Core Attack Logic)
// Black Hole Receive Callback Function bool BlackholeReceiveCallback(Ptr<NetDevice> device, Ptr<const Packet> packet, uint16_t protocol, const Address &sender) { // Log the dropped packet for analysis NS_LOG_UNCOND("‚ö´ Blackhole node dropped packet of size " << packet->GetSize() << " bytes"); // Return true to indicate packet has been "handled" // This prevents packet from being forwarded to upper layers return true; // Packet consumed (dropped) } // Configure Node 2 as the Black Hole attacker Ptr<NetDevice> blackholeDevice = devices.Get(2); blackholeDevice->SetReceiveCallback(MakeCallback(&BlackholeReceiveCallback));

4. Technical Insights

4.1 AODV Protocol Vulnerability

The Ad-hoc On-Demand Distance Vector (AODV) routing protocol is susceptible to Black Hole attacks due to:

  • Trust-based Operation: AODV assumes all nodes cooperate honestly in route discovery and maintenance
  • No Authentication: Route Reply (RREP) messages are not cryptographically verified
  • Reactive Nature: Routes are established on-demand without prior verification of node behavior
  • Lack of Monitoring: No built-in mechanism to detect packet forwarding failures

4.2 Attack Mechanism Breakdown

Attack Flow Diagram (Sequential)
1. Node 0 generates UDP packet destined for Node 4 ‚Üì 2. AODV initiates Route Discovery (RREQ broadcast) ‚Üì 3. Node 2 (Blackhole) receives RREQ and forwards it ‚Üì 4. Node 4 sends Route Reply (RREP) back through Node 2 ‚Üì 5. Route established: 0 ‚Üí 1 ‚Üí 2 ‚Üí 3 ‚Üí 4 ‚Üì 6. Node 0 starts sending data packets ‚Üì 7. ‚ö´ Node 2 DROPS all packets (Black Hole activated) ‚Üì 8. Node 4 receives NOTHING (Communication disrupted)

4.3 Why This Attack is Effective

  • Silent Operation: The malicious node participates normally in routing until data transmission begins
  • No Error Signals: Dropped packets generate no ICMP errors or routing updates
  • Resource Exhaustion: Source continues sending packets, wasting bandwidth and energy
  • Difficult Detection: Appears as normal packet loss or congestion to upper layers

5.2 Detection Algorithm Implementation

Approach: Watchdog-based intrusion detection system

Proposed Detection Mechanism

  • Monitor Mode: Each node monitors neighbors' forwarding behavior
  • Threshold-based: If forwarding ratio drops below 80%, flag as suspicious
  • Cooperative Detection: Nodes share blacklists via secure channels
  • Route Avoidance: AODV modified to exclude blacklisted nodes from routes

5.3 Advanced Attack Variants

Gray Hole Attack: Selective packet dropping (more stealthy)

Gray Hole Implementation (Selective Dropping)
bool GrayHoleReceiveCallback(Ptr<NetDevice> dev, Ptr<const Packet> pkt, uint16_t protocol, const Address &addr) { // Drop 50% of packets randomly (harder to detect) if (rand() % 100 < 50) { NS_LOG_UNCOND("Gray Hole: Dropped packet"); return true; // Drop } return false; // Forward normally }

Collaborative Black Hole: Multiple malicious nodes coordinating

  • Multiple nodes form a malicious coalition
  • Cooperate to send fake routing information
  • Increases attack success rate significantly
  • More difficult to detect and mitigate

7. Conclusion

This research successfully demonstrated the vulnerability of AODV-based ad-hoc networks to Black Hole attacks. The simulation clearly illustrated how a single malicious node can completely disrupt network communication by silently dropping packets while appearing to participate in routing protocols.

Key Findings:

  • Critical Vulnerability: AODV's trust-based architecture makes it susceptible to node misbehavior
  • Complete DoS: A strategically placed Black Hole can achieve 100% packet loss
  • Detection Challenge: The attack operates silently without generating routing errors
  • Network Impact: All Quality of Service (QoS) metrics severely degraded

Research Contributions:

  • Practical implementation of Black Hole attack in ns-3 simulator
  • Visual demonstration using NetAnim for educational purposes
  • Comprehensive documentation of setup and execution process
  • Foundation for future detection and mitigation research

Real-World Implications

This research has significant implications for IoT networks, vehicular ad-hoc networks (VANETs), and military tactical communications where AODV is commonly deployed. Organizations implementing such networks must consider:

  • Implementing node authentication mechanisms
  • Deploying intrusion detection systems
  • Using redundant communication paths
  • Regular security audits of network nodes
